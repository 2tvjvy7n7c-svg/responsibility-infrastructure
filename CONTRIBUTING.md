# Feedback and Contributions

This repository welcomes serious criticism, corrections, counterarguments, and relevant research.

## Preferred feedback

Open a GitHub Issue when proposing:

- a factual correction,
- a source that supports or challenges a claim,
- a logical objection,
- a counterexample,
- a terminology clarification,
- or an editorial correction.

A useful submission should identify:

1. the exact passage or claim,
2. the proposed correction or objection,
3. the reasoning,
4. any supporting evidence or source,
5. and whether the issue affects the thesis or only its presentation.

## Pull requests

Do not submit a pull request that silently rewrites the author's thesis or article.

For substantive changes, open an Issue first so the reasoning can be reviewed and preserved.

Minor corrections such as typographical errors may be submitted through a pull request.

## Discussions

Broader discussion of responsibility infrastructure, governed AI-assisted work, process study, authority, evidence, and accountability may be opened through GitHub Discussions.

Do not use this repository to request or speculate about proprietary Doghouse source code, security mechanisms, prompts, schemas, or internal architecture.
